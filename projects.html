Box Sorting Robotics

This is a project i have been working on for a course called 2D & 3D Animation at Nanyang Technological University in Singapore. It is a robot arm simulation that tries to sort boxes based on color while the user disrupts with the mouse pointer

The robot arm movement is simulated using inverse kinematics. Each delta movement is found with an pseudo inverse jacobian algorithm with damping. The arm has been made stiff and the target position for each segment is manipulated to avoid "uncomfortable" positions. The algorithm was first tested in a pure javascript 2D version which can be found here, and was later ported to this improved 3D version using webGL through the Three.js framework. Apart from that, Cannon.js is used for the box physics and Sylvester.js is used for matrix operations.

Project can be found on Github and this demopage

Freakolution

Freakolution is a 2D/3D hybrid multiplayer game made by me and three team members in Unity3D for a Game Design course at Nanyang Technological University (NTU) in singapore

The game is a wave-survival game with a top-down perspective. Freakolution won first prize in the MAGIC game challenge competition at NTU for outstanding performance. It can be played by four people on the same screen (not split screen) and was intended for a console platform. The game was written in C# and it gave me good base knowledge for the language.

MMDG

Massive Multiplayer Dome Game (MMDG) is a system that allows a developer to connect their program to a server (MMDG-server), so that players can connect and control it through their phone.

The system was created by me and six other team members for our under-graduate examination project. Our customer was Visualization Center C in Norrköping Sweden, which hosts a large-scale arena for public visits and events including media labs, interactive exhibitions and an immersive 3D fulldome theatre. Our assignment was to make it possible for an audience at the dome theatre to connect to an application through their phone's web-browser, displaying a customizable interface, so that they can interact with the show. It was also intended to be used as a pre-show entertainment for the audience to be able to play a small and simple multiplayer game in the dome while waiting for the show to start.

A developer can customize the GUI that displays on the users phone with a XML configuration file. When users connect to the server the GUI is rendered and a websocket connection is established. Then a TCP connection needs to be established between the developers application and the MMDG-server. If it is used in the dome-theatre, a MMDG-core class written in C++ can be used for this connection. When the user interacts with the developer-defined buttons, messages are sent to the application.

the proof-of-concept-game we created was called RobberCop. It allowed around one hundred concurrent clients to play the game at once on the same screen

attempt to show system design, and communication between modules.

This project really introduced us to many aspects of software development. Working with a real customer with a big team (eight people) and following a development methodology (Scrum). We had to learn about internet protocols, server programming and writing programs for the dome-theatre. Java was used for the server, Javascript and HTML/CSS for the GUI, and OpenGL with a framework called SGCT for the game.

Project can be found on Github

B-spline Editor

A simple B-spline editor made by me for a 3D Modelling course at Nanyang Technological University (NTU) in singapore

The editor is not very advanced or intuitive unfortunatly. The assignment was to be able to render a B-spline with a uniform algorithm and a Adaptive algorithm from a text file that specifies:

  • The degree of B-spline
  • The number of Control Points
  • Knot sequence in 1 row
  • Control points, 1 point per row

When this was done, I decided to further develop the the program with another algorithm called DeBoor algorithm and improve the user interface a bit. Now you can create B-splines directly in the Canvas and move the control points around. There is still no way to specify the knot sequence from the interface, so it only renders uniform B-splines

Project can be found on Github and demopage
